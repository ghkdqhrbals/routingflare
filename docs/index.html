routingflare

macOS menu bar app for exposing local development ports through Cloudflare Tunnel. Add Quick URL or DNS routes, restrict inbound access, and update from the app.

로컬 dev 서버를 메뉴바에서 바로 외부 오픈.

routingflare는 로컬 필터링 프록시를 통해 Cloudflare Tunnel을 열고, 라우팅 리스트를 항상 상단에 보여줍니다.

Quick URL

trycloudflare.com 임시 주소를 받아 포트와 path를 로컬 서버로 라우팅합니다.

DNS routes

기존 Cloudflare named tunnel 설정으로 hostname, path, local port를 묶어 관리합니다.

Security

Inbound IP/CIDR allowlist와 optional auth header secret을 재시작 없이 적용합니다.

Independent routes

Quick URL, DNS, Security, Logs 탭을 바꿔도 라우팅 테이블은 독립적으로 유지됩니다.

Logs

cloudflared output, public URL 파싱, proxy allow/deny 이벤트를 한 탭에서 확인합니다.

Updates

About 팝업에서 GitHub Pages 소개 페이지를 열고, 최신 DMG를 체크/다운로드할 수 있습니다.

Toolbar screenshots.

Quick URL, DNS, Security 탭이 같은 route table을 공유하고, 각 탭은 추가/보안 설정만 담당합니다.

동작 흐름.

1 로컬 서버 실행

127.0.0.1:8989 같은 dev 서버를 먼저 켭니다.

2 route 추가

Quick URL은 port/path, DNS는 hostname/port/path를 추가합니다.

3 Start

상단 route list 전체를 기준으로 필요한 tunnel/proxy를 실행합니다.

4 접근 제한

Security 탭에서 IP allowlist와 auth header secret을 관리합니다.
